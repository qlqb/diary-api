## 1. 이 목업에서 가져올 것

- **세로 배치 순서 (척추)**: 지난 계획 정리 → 입력 → 오늘 리스트 → 오늘 마무리.
  하루의 시간 순서(아침 정리 → 계획 → 실행 → 밤 마무리)와 일치.
  1차-C에서 하루 마무리 카드가 생기면 맨 아래 위치가 맞다.
- **리스트 행 구성**: 체크 ○ + 제목 + 우선순위 뱃지 + 시간(오른쪽 정렬).
- **사이드바 네비게이션 형태**: 최종형에 가깝다. 단, 1차-D(오늘 화면 중심 재편)에서
  메뉴 구성과 함께 결정할 일이며 지금 탭 구조를 갈아엎을 이유는 아니다.
- **pending [더 보기] 접기 패턴**: pending이 많이 쌓일 때를 위한 후속 개선 후보.

## 2. 이 목업에서 따라 하면 안 되는 것 (확정 결정과 충돌)

| 목업 요소 | 충돌하는 결정 |
|---|---|
| "어제 미완료" 뱃지 | 피할 문구 원칙 ("미완료/실패/밀린" 금지). 최우선 교정 대상 |
| [작게 줄이기] 버튼 | 줄이기는 pending 카드 UI에서 제외 확정 (빠른 판단 목적) |
| [내일로 이동] | 우리 결정은 "오늘로 이동". 내일로 이동은 정리가 아니라 연기이며 pending을 이월시킴 |
| AI 추천 패널 (화면 우측 1/3) | 제외 목록 명시 + 백엔드 부재 |
| 오늘 마무리 카드 (감정/한 줄 회고) | 1차-C~D 영역. 백엔드 부재. 위치 참고만 |
| 사이드바 상시 "정리함" 메뉴 | 보류함 상시 노출 금지 방향 (나중에 할 일 목록으로 오용 → 수집함과 구분 붕괴). 보류함 UI는 주간 회고 등 특정 시점 노출 검토 |
| 사이드바 "회고" 메뉴 | WeeklyReview는 제외 목록 |
| "오늘의 핵심 (MUST)" 영어 병기 | 한글 라벨만 사용 확정. 내부 enum을 사용자에게 노출하지 않음 |

## 3. 우선순위 한글 라벨 (확정)

| enum | UI 라벨 (입력 폼) | 리스트 뱃지 |
|---|---|---|
| MUST | 🎯 오늘의 핵심 | 🎯 핵심 |
| SHOULD | 🌱 하면 좋은 것 (기본값) | 표시 안 함 (기본값이라 조용히) |
| OPTIONAL | ☕ 여유 있으면 | ☕ 여유 |

원칙: 등급("필수/보통/낮음")이 아니라 성격을 말하는 표현. 압박 어감 금지.
