# 02. Feature Structure

## 1. 기능 계층 구조

diary-app의 기능은 같은 무게로 나열하지 않는다.

기능은 다음 4계층으로 나눈다.

```text
1계층: 기록
2계층: 정리
3계층: 실행
4계층: 분석
```

## 2. 기록 계층

사용자가 가장 먼저, 가장 자주 사용하는 입력 기능이다.

- 일기
- 회고
- 감정 기록
- 수면 기록
- 공부 기록
- 지출 기록
- 실행 결과 기록

기록 계층의 목적은 사용자의 하루와 상태를 부담 없이 남기는 것이다.

초기 단계에서는 분석보다 기록의 마찰을 낮추는 것이 중요하다.

## 3. 정리 계층

기록 내용을 바탕으로 사용자가 반복적으로 신경 쓰는 문제나 주제를 모아두는 영역이다.

우선 표현은 `정리함`이다. 과거 문서에서 사용한 `문제함`은 같은 계열의 낡은 표현으로 보고, 새 문서와 화면에서는 정리함을 우선한다.

- 정리함 직접 등록
- 정리함 목록/상세/수정/삭제
- 정리함과 Todo 연결
- 정리함과 일기 연결
- AI 문제 후보 추출

AI가 만든 문제 후보는 자동 저장되지 않는다. 사용자가 선택한 항목만 정리함에 저장된다.

## 4. 실행 계층

정리된 문제나 오늘 해야 할 일을 실제 행동과 하루 운영으로 바꾸는 영역이다.

- Todo
- ScheduleBlock
- DailyPlan
- 루틴
- 오늘 해볼 것

역할은 다음처럼 구분한다.

```text
Todo = 완료 가능한 구체 행동
ScheduleBlock = 오늘 안에서 언제 또는 어떤 순서로 할지 배치한 행동 단위
DailyPlan = 오늘 하루의 보기 방식과 운영 상태
```

실행 계층은 거창한 계획보다 오늘 할 수 있는 작은 행동과 조정을 우선한다.

## 5. 분석 계층

기록, 정리, 실행 데이터를 연결해 반복 패턴을 보여주는 영역이다.

- 이벤트 집계 기반 주간 회고
- AI 주간 요약
- 감정/수면 quick log 집계
- 실행 결과 분석
- 교차 분석

AI 주간 요약은 1.5차 범위다. 사용자가 명시적으로 요청할 때만 실행하며, 일기 원문 전체를 기본 전송하지 않는다.

교차 분석은 확장 범위다. 예시는 다음과 같다.

- 컨디션 태그 × 이월률
- 수면 × 실행률
- 감정 × 지출
- 루틴 × Todo 완료율

## 6. 전체 기능 구조

```text
기록
  - 일기
  - 회고
  - 감정 기록
  - 수면 기록
  - 공부 기록
  - 지출 기록
  - 실행 결과 기록

정리
  - 정리함
  - 정리함-Todo 연결
  - 정리함-일기 연결
  - AI 문제 후보

실행
  - Todo
  - ScheduleBlock
  - DailyPlan
  - 루틴
  - 오늘 해볼 것

분석
  - 주간 회고
  - AI 주간 요약
  - quick log 집계
  - 실행 이벤트 집계
  - 교차 분석
```

## 7. 핵심 흐름

```text
기록한다
→ 필요하면 정리함에 모은다
→ 오늘 할 작은 행동으로 바꾼다
→ DailyPlan과 ScheduleBlock으로 운영한다
→ 완료/이동/축소/보류 이벤트를 남긴다
→ 주간 회고에서 반복 패턴을 확인한다
```
