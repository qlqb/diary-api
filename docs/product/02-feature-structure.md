# 02. Feature Structure

## 1. 기능 계층 구조

이 앱의 기능은 같은 무게로 나열하지 않는다.

기능은 다음 4계층으로 나눈다.

```text
1계층: 기록
2계층: 정리
3계층: 실행
4계층: 분석
```

## 2. 1계층: 기록 계층

사용자가 가장 먼저, 가장 자주 사용하는 기본 입력 기능이다.

- 일기
- 회고
- 감정 기록
- 수면 기록
- 공부 기록
- 지출 기록

이 계층의 목적은 사용자가 자신의 하루와 상태를 부담 없이 남기는 것이다.

이 단계에서는 분석보다 기록의 편의성이 중요하다.

## 3. 2계층: 정리 계층

기록된 내용을 바탕으로 사용자가 반복적으로 관리하고 싶은 문제를 모아두는 영역이다.

- 문제함 / 정리함
- 목표관리
- AI 문제 후보 추출

예시:

```text
일기:
요즘 영어 공부가 계속 밀리고, 피곤할 때마다 편의점에서 돈을 쓰는 것 같다.

AI 문제 후보:
1. 영어 공부 루틴이 지속되지 않음
2. 피곤할 때 편의점 지출이 반복됨
3. 수면 부족이 공부와 소비에 영향을 줌
```

단, 이 문제들은 자동 저장되지 않는다.

사용자가 선택한 항목만 문제함에 저장된다.

## 4. 3계층: 실행 계층

정리된 문제를 실제 행동과 시간 배치로 바꾸는 영역이다.

- Todo
- 루틴
- 오늘 해볼 것
- 하루 계획
- 시간 블록
- 알림

이 계층의 목적은 거창한 계획을 만드는 것이 아니라, 오늘 또는 내일 할 수 있는 작은 행동을 만드는 것이다.

계획 시스템의 상세 구조는 `03-planning-system.md`에서 관리한다.

예시:

```text
문제:
피곤할 때 편의점 지출이 반복됨

해볼 만한 행동:
1. 알바 끝나고 바로 편의점 가지 말고 집에 있는 간식 먼저 확인하기
2. 편의점 예산을 하루 5,000원으로 제한하기
3. 수면 부족한 날은 배달/편의점 지출을 기록만 하기
```

## 5. 4계층: 분석 계층

기록, 문제, todo, 루틴, 수면, 공부, 지출 데이터를 연결해 패턴을 보여주는 영역이다.

- 통계
- AI 분석
- 소비 회고
- 공부 회고
- 수면-감정 분석
- 루틴-지출 분석
- 주간 회고
- 월간 회고

이 계층은 MVP 초기에는 너무 크게 만들지 않는다.

데이터가 쌓인 뒤에 의미가 생기기 때문이다.

## 6. 전체 기능 구조

```text
기록
  - 일기
  - 회고
  - 감정 기록
  - 수면 기록
  - 공부 기록
  - 지출 기록

정리
  - 문제함 / 정리함
  - 목표관리
  - AI 문제 후보 추출

실행
  - Todo
  - 루틴
  - 오늘 해볼 것
  - 하루 계획
  - 시간 블록
  - 알림

분석
  - 통계
  - AI 패턴 분석
  - 소비 회고
  - 공부 회고
  - 주간 회고
  - 월간 회고
```

## 7. 핵심 흐름

이 앱의 핵심 흐름은 다음과 같다.

```text
기록한다
→ 원하면 정리한다
→ 필요한 문제만 저장한다
→ 작은 행동으로 연결한다
→ 계획에 배치한다
→ 실행한다
→ 다시 기록한다
→ 반복 패턴을 확인한다
```
