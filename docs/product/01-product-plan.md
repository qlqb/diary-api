# 01. Product Plan

## 1. 앱의 최종 방향

이 앱은 단순 일기장, todo 앱, 루틴 앱, 가계부 앱이 아니다.

사용자가 가진 여러 문제를 한곳에서 기록하고, 정리하고, 실행 가능한 작은 행동으로 연결하며, AI가 사용자의 기록과 패턴을 바탕으로 보조 분석을 제공하는 자기관리 웹 애플리케이션이다.

핵심 목적은 사용자를 더 바쁘게 만드는 것이 아니라, 흩어진 문제를 한곳에 모아 복잡도를 낮추는 것이다.

## 2. 핵심 포지션

이 앱의 포지션은 다음과 같다.

> 기록 기반 문제 정리형 자기관리 앱

더 구체적으로는 다음과 같다.

> 일기, 루틴, 공부, 수면, 돈, 감정 데이터를 한곳에 기록하고, AI가 이 데이터 사이의 관계를 분석해 사용자의 반복 문제와 다음 행동 후보를 제안하는 앱

단, AI는 사용자의 판단을 대신하지 않는다.

AI는 기록을 요약하고, 문제 후보를 제안하고, 행동 후보를 보여주는 보조 역할만 한다.

## 3. 핵심 UX 원칙

이 앱은 사용자를 압박하는 생산성 앱이 아니라, 부담을 줄이는 자기정리 앱이어야 한다.

핵심 UX 원칙은 다음과 같다.

- 기록은 자유롭게
- 분석은 요청할 때만
- 추천은 작은 후보로
- 실행은 사용자가 선택

세부 원칙은 다음과 같다.

1. 사용자가 일기를 저장했다고 해서 AI 분석을 자동 실행하지 않는다.
2. AI는 사용자가 요청했을 때만 요약, 문제 후보, 행동 후보를 제공한다.
3. AI가 만든 문제, todo, 소비 기록, 계획은 바로 저장하지 않고 후보로 보여준다.
4. 사용자가 선택하거나 적용한 항목만 최종 저장한다.
5. “해야 할 일”보다 “해볼 만한 행동”이라는 표현을 사용한다.
6. 실패나 미달보다 반복 패턴과 다음 행동을 보여준다.
7. 사용자가 AI 후보를 수정한 뒤 적용할 수 있어야 한다.

## 4. 올인원 전략

사용자가 원하는 것은 기능이 많은 앱이 아니라, 자기 문제를 덜 복잡하게 만드는 앱이다.

따라서 올인원 앱을 만들더라도 모든 기능을 같은 깊이로 만들면 안 된다.

이 앱의 올인원 전략은 다음과 같다.

> 모든 기능을 다 잘하는 앱이 아니라, 여러 생활 데이터를 하나의 문제 정리 흐름으로 연결하는 앱

즉, 일기, 공부, 수면, 돈, 감정, 루틴은 각각 독립적인 전문 앱처럼 깊게 만들지 않는다.

대신 이 데이터들이 서로 어떤 영향을 주는지 보여주는 데 집중한다.

예시 흐름:

```text
수면 부족
→ 공부 집중 실패
→ 스트레스 증가
→ 편의점/배달 지출 증가
→ 자기비난
→ 다음 날 루틴 실패
```

이런 연결을 보여주는 것이 이 앱의 핵심 가치다.

## 5. 최종 방향 문장

이 앱은 사용자의 모든 문제를 한 번에 해결하는 올인원 앱을 목표로 한다.

하지만 모든 기능을 전문 앱처럼 깊게 만드는 것이 아니라, 사용자의 생활 데이터를 하나의 흐름으로 연결하는 방식으로 올인원을 구현한다.

최종 방향은 다음과 같다.

> 일기, 감정, 수면, 공부, 돈, 루틴 데이터를 한곳에 기록하고, AI가 이 데이터 사이의 관계를 분석해 사용자의 반복 문제와 작은 행동 후보를 제안하는 자기관리 웹 애플리케이션

돈관리 기능은 다음처럼 정의한다.

> 돈을 꼼꼼히 관리하는 가계부가 아니라, 내 감정과 생활 패턴 속에서 왜 돈을 쓰는지 돌아보게 하는 소비 회고 도구

따라서 이 앱의 핵심 차별점은 다음이다.

> 문제별 전문 관리 기능이 아니라, 문제들이 서로 연결되는 방식을 보여주는 올인원 자기관리 구조
