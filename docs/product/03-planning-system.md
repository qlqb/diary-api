# 03. Planning System

## 1. 계획 시스템의 목적

계획 시스템은 단순 캘린더나 시간표 기능이 아니다.

이 기능은 사용자의 목표, 문제함, Todo, 루틴, 회고를 연결해 실제 실행 가능한 흐름으로 만드는 상위 구조이다.

핵심 구조는 다음과 같다.

```text
연간 목표
→ 월간 계획
→ 주간 계획
→ 하루 계획
→ Todo
→ 시간 블록
→ 실행
→ 회고
```

## 2. 계층별 역할

| 계층 | 역할 |
|---|---|
| 연간 목표 | 올해 집중할 방향과 큰 목표 |
| 월간 계획 | 이번 달 집중 영역과 실행 후보 |
| 주간 계획 | 이번 주 우선순위와 작업 배분 |
| 하루 계획 | 오늘 할 일을 실제 실행 단위로 정리 |
| Todo | 실제 행동 항목 |
| 시간 블록 | 몇 시부터 몇 시까지 할지 배치 |

## 3. 연간 목표

연간 목표는 사용자가 올해 집중하고 싶은 큰 방향과 개선하고 싶은 문제를 정리하는 최상위 목표이다.

예시:

```text
2026년 목표
- 개발 실력 올리기
- 영어 공부 습관 만들기
- 수면 패턴 안정화
- 소비 습관 개선
- 감정 기록 습관 만들기
```

연간 목표는 하루 단위의 세부 행동을 직접 관리하지 않는다.

연간 목표는 월간 계획과 주간 계획의 기준점으로 사용된다.

## 4. 월간 계획

월간 계획은 연간 목표를 한 달 단위의 집중 영역과 실행 후보로 나누는 계획이다.

예시:

```text
2026년 7월 월간 계획

이번 달 핵심 주제:
- diary-app MVP 완성
- 자료구조 연결 리스트 복습
- 영어 루틴 유지
- 소비 패턴 기록하기

이번 달 목표:
1. diary-app 일기 CRUD 완성
2. Todo 기능 완성
3. 자료구조 연결 리스트 문제 20개 풀기
4. 영어 주 4회 공부
```

월간 계획은 시간표가 아니라 방향과 집중 영역을 정리하는 문서에 가깝다.

## 5. 주간 계획

주간 계획은 월간 계획을 이번 주 우선순위와 할 일 후보로 나누는 중간 계획이다.

예시:

```text
7월 1주차 주간 계획

이번 주 우선순위:
1. diary-app Todo 테이블 설계
2. 일기 CRUD 마무리
3. 자료구조 단순 연결 리스트 복습
4. 영어 4회 학습

이번 주 할 일:
- Todo ERD 정리
- Todo API 설계
- ScheduleBlock 초안 설계
- 자료구조 연결 리스트 삽입/삭제 구현
- 영어 Unit 3개 진행
```

주간 계획은 Todo 후보를 만들거나 요일별로 대략 배분하는 역할을 한다.

## 6. 하루 계획

하루 계획은 오늘 할 일을 실제 시간대에 배치하는 실행 계획이다.

예시:

```text
2026-07-01 하루 계획

10:00 - 10:40 자료구조 복습
11:00 - 12:00 diary-app 요구사항 정리
14:00 - 15:30 diary-app 개발
16:00 - 16:30 영어 복습
22:30 - 22:40 오늘 회고
```

하루 계획은 시간 블록을 포함할 수 있다.

각 시간 블록은 Todo와 연결될 수 있다.

```text
Todo = 무엇을 할지
ScheduleBlock = 언제 할지
```

## 7. AI 계획 생성

사용자는 각 계획을 직접 작성할 수 있으며, AI의 도움을 받아 계획 후보를 생성할 수도 있다.

AI는 다음 후보를 생성할 수 있다.

- AI 연간 목표 후보
- AI 월간 계획 후보
- AI 주간 계획 후보
- AI 하루 계획 후보
- AI 시간 블록 후보

단, AI가 생성한 목표와 계획은 자동 저장되지 않는다.

항상 다음 흐름을 따른다.

```text
AI 생성
→ 후보 미리보기
→ 사용자 수정 가능
→ 적용
→ 저장
```

## 8. AI 생성 후 사용자 수정 처리

AI가 생성한 계획 또는 시간 블록을 사용자가 수정한 뒤 적용한 경우, 해당 데이터는 다음처럼 저장한다.

```text
origin_type = AI_GENERATED
modified_after_creation = true
```

직접 만든 경우는 다음처럼 저장한다.

```text
origin_type = MANUAL
modified_after_creation = false
```

이렇게 해야 다음 분석이 가능하다.

- AI가 만든 계획을 사용자가 얼마나 수정하는가
- AI 계획 후보를 그대로 적용했을 때 완료율은 어떤가
- 사용자가 수정한 AI 계획은 원본 AI 계획보다 실행률이 높은가

## 9. MVP 적용 범위

계획 시스템 전체를 MVP에 모두 넣지 않는다.

권장 순서는 다음과 같다.

```text
1차 MVP:
- 하루 계획
- 시간 블록 직접 생성
- Todo를 시간 블록에 연결

2차:
- AI 하루 계획 후보 생성
- AI 후보 수정 후 적용

3차:
- 주간 계획

4차:
- 월간 계획

5차:
- 연간 목표
```

기획 문서에는 전체 구조를 포함하되, 구현은 단계적으로 진행한다.
